## 项目概述

### 什么是 CutClaw？

CutClaw 是一个**端到端的智能视频编辑系统**，专门用于将数小时的原始素材和音乐自动剪辑成电影级别的蒙太奇视频。

**核心价值：**
- 🎬 **一键分解**：将长视频自动分解为结构化、可搜索的素材
- 🎯 **指令控制**：只需一条文本指令即可控制编辑风格
- 🎵 **音乐同步**：自动提取音乐节拍，实现节奏感知的剪辑
- 🤖 **多智能体协作**：Screenwriter（编剧）、Editor（编辑）、Reviewer（审核）协同工作

### 应用场景

1. **社交媒体内容创作**：快速生成适合 TikTok/Instagram/YouTube 的短视频
2. **电影预告片制作**：从长电影中提取精彩片段
3. **音乐视频剪辑**：根据音乐节奏自动剪辑画面
4. **个人 vlog 编辑**：自动化日常视频剪辑流程

### 设计模式

CutClaw 采用了多种设计模式：

1. **多智能体模式 (Multi-Agent Pattern)**
   - Screenwriter、Editor、Reviewer 各司其职
   - 类似微服务架构中的服务拆分

2. **工厂模式 (Factory Pattern)**
   - `_create_decord_reader()` 创建视频阅读器
   - 支持 GPU/CPU 自动回退

3. **策略模式 (Strategy Pattern)**
   - 多种音频检测方法（downbeat, pitch, mel_energy）
   - 可配置的 ASR 后端（litellm / whisper_cpp）

4. **重试模式 (Retry Pattern)**
   - API 调用失败时自动重试（指数退避）
   - 使用 `tenacity` 库实现

5. **线程局部存储模式 (ThreadLocal Pattern)**
   - 每个线程独立的视频阅读器
   - 避免并发冲突

### 依赖库详解

#### 1. LiteLLM
- **作用**：统一的 LLM API 网关
- **支持模型**：OpenAI GPT、Anthropic Claude、Google Gemini、MiniMax、Kimi 等
- **优势**：单一接口调用不同厂商的模型

#### 2. decord
- **作用**：高性能视频帧提取
- **特点**：支持 GPU 加速（NVDEC）
- **性能**：比 OpenCV 快 10-50 倍

#### 3. madmom
- **作用**：音乐信息检索
- **功能**：节拍检测、能量分析、音高提取
- **应用**：实现音乐感知的视频剪辑

#### 4. PySceneDetect
- **作用**：自动检测视频场景转换
- **算法**：自适应阈值检测
- **输出**：场景边界时间码列表

## 工作流程

### 完整处理流程

```
1. 用户上传文件
   ├─ 视频文件 (.mp4/.mkv)
   ├─ 音频文件 (.mp3/.wav)
   └─ 编辑指令 (文本)

2. 预处理阶段（首次运行，耗时较长）
   ├─ 视频分解
   │  ├─ 场景检测 (PySceneDetect)
   │  ├─ 帧采样 (decord, 2 FPS)
   │  ├─ VLM 内容描述 (Gemini/Qwen)
   │  └─ 角色识别
   ├─ 音频分析
   │  ├─ ASR 字幕识别 (litellm/whisper)
   │  ├─ 节拍检测 (madmom downbeat)
   │  ├─ 能量分析 (mel_energy)
   │  └─ 音高提取 (pitch)
   └─ 缓存结果到 Output/<video_audio>/

3. 智能体协作阶段
   ├─ Screenwriter Agent
   │  ├─ 读取视频摘要和音乐结构
   │  ├─ 识别 Hook 对话
   │  └─ 生成 shot_plan.json
   │
   ├─ Editor Agent (循环执行)
   │  ├─ 接收 shot_plan
   │  ├─ 调用工具：
   │  │  ├─ semantic_neighborhood_retrieval()
   │  │  └─ fine_grained_shot_trimming()
   │  └─ 生成 shot_point.json
   │
   └─ Reviewer Agent
      ├─ 验证时间范围
      ├─ 检查重复片段
      ├─ VLM 面部质量检测
      └─ 美学质量评分

4. 渲染输出阶段
   ├─ 读取 shot_plan.json 和 shot_point.json
   ├─ MoviePy 视频合成
   ├─ 智能裁剪 (9:16, 16:9 等)
   ├─ 添加字幕和标签
   └─ 输出 final.mp4
```

### 并行处理优化

CutClaw 使用多线程/多进程优化性能：

1. **视频帧处理**：`ThreadPoolExecutor` 并行处理多个镜头
2. **场景检测**：`ProcessPoolExecutor` 并行检测不同视频段
3. **API 调用**：`asyncio.Semaphore` 限制并发请求数
4. **线程局部存储**：每个线程独立的视频阅读器，避免锁竞争

**Java 类比**：
```java
// Python 的 ThreadPoolExecutor
ExecutorService executor = Executors.newFixedThreadPool(4);

// Python 的 asyncio.Semaphore
Semaphore semaphore = new Semaphore(5);

// Python 的 ThreadLocal
ThreadLocal<VideoReader> threadLocal = new ThreadLocal<>();
```

## 关键概念

### 1. Shot Plan vs Shot Point

**Shot Plan (镜头计划)**
- **生成者**：Screenwriter Agent
- **内容**：抽象的镜头序列规划
- **格式**：JSON，包含镜头类型、时长、情感等元数据
- **示例**：
```json
{
  "shots": [
    {
      "type": "close_up",
      "duration": 3.5,
      "emotion": "intense",
      "description": "Joker's face close-up"
    }
  ]
}
```

**Shot Point (镜头时间点)**
- **生成者**：Editor Agent
- **内容**：具体的时间戳和裁剪信息
- **格式**：JSON，包含起始时间、结束时间、裁剪区域
- **示例**：
```json
{
  "shots": [
    {
      "start_time": "00:12:34.5",
      "end_time": "00:12:38.0",
      "crop_box": {"x": 100, "y": 50, "width": 400, "height": 600}
    }
  ]
}
```

### 2. 语义邻域检索 (Semantic Neighborhood Retrieval)

**目的**：找到与当前镜头语义相关的其他镜头

**工作原理**：
1. 计算当前镜头的语义向量（通过 VLM 描述）
2. 在视频摘要数据库中搜索相似镜头
3. 返回最相关的 N 个候选镜头

**Java 类比**：类似 Elasticsearch 的相似度搜索

### 3. 精细镜头修剪 (Fine-Grained Shot Trimming)

**目的**：精确调整镜头的起始和结束时间

**工作流程**：
1. 接收初始时间范围
2. 逐帧分析主角出现情况
3. 根据音乐节拍微调边界
4. 确保平滑过渡

**Java 类比**：类似视频编辑软件的时间线修剪功能

### 4. 音乐感知剪辑 (Music-Aware Editing)

**关键技术**：
- **Downbeat Detection**：检测强拍（每小节的第一个拍子）
- **Mel Energy**：旋律能量分析
- **Pitch Tracking**：音高跟踪

**应用**：
- 在强拍处切换镜头
- 根据能量变化调整剪辑节奏
- 匹配画面动作与音乐节拍

**Java:**

```java
ExecutorService executor = Executors.newFixedThreadPool(4);

List<Result> results = futures.stream()
```

## 常见问题 (FAQ)

### Q1: 如何处理非常大的视频文件？

**A**: 
1. 使用 `max_minutes` 参数限制处理时长
2. 降低 `VIDEO_FPS` 减少采样帧数
3. 降低 `VIDEO_RESOLUTION` 减小分辨率
4. 启用 GPU 加速解码

```python
--config.MAX_MINUTES 10  # 只处理前 10 分钟
--config.VIDEO_FPS 1     # 每秒 1 帧
--config.VIDEO_RESOLUTION 480  # 480p
```

### Q2: 如何更换 LLM 提供商？

**A**: 修改 `config.py` 中的模型名称：

```python
# 使用 Claude
AGENT_MODEL = "anthropic/claude-3-opus-20240229"

# 使用 Gemini
AGENT_MODEL = "google/gemini-1.5-pro"

# 使用 MiniMax
AGENT_MODEL = "minimax/minimax-m2.7"
```

确保在 `.env` 中配置对应的 API 密钥。

### Q3: 如何自定义编辑风格？

**A**: 修改 `INSTRUCTION` 参数：

```python
# 快节奏动作蒙太奇
INSTRUCTION = "Create a fast-paced action montage with quick cuts"

# 慢节奏情感叙事
INSTRUCTION = "Create a slow-paced emotional narrative with long takes"

# 音乐视频风格
INSTRUCTION = "Edit like a music video, sync cuts to beats"
```

### Q4: 如何添加自定义字体？

**A**: 
1. 将字体文件放入 `resource/font/` 目录
2. 在 `render/render_video.py` 中指定字体路径

```python
font_path = "resource/font/MyCustomFont.ttf"
```

### Q5: 如何调试智能体的决策过程？

**A**: 
1. 查看生成的 JSON 文件：
   - `Output/<video_audio>/shot_plan_*.json`
   - `Output/<video_audio>/shot_point_*.json`

2. 启用详细日志：
   ```python
   --config.DEBUG True
   --config.LOG_LEVEL DEBUG
   ```

3. 检查 VLM 响应：
   ```python
   # 在 core.py 中添加打印语句
   ```
